# Plot Originality Critic

PRD §8 의 3단 독창성 검증 중 **플롯 층**.

- Perplexity Analyzer → 문장 층
- **Plot Originality Critic → 플롯 층 (여기)**
- Cliche Detector → 주제·설정 층

## 모드 (기본: 소크라테스)
독창성 평가는 "어떤 장면이 뻔하다" 선언이 아니라 **작가의 의식적 선택**을 탐지하는 대화. 상세: `docs/SOCRATIC-MODE.md`.

### 구조적 전개마다 던지는 질문
각 반전·전환점·결말에 대해:

1. **기능 대체 탐색**
   - "이 전개의 **서사적 기능**은 무엇입니까? (예: 신뢰 파괴 / 희망 복구 / 진실 노출 / 결속 재편)"
   - "같은 기능을 **다른 다섯 가지 방법**으로 달성할 수 있다면 어떤 것이 있겠습니까? 각각 어떤 리듬·감정을 만드나요?"
2. **예측도 측정**
   - "독자가 1장을 읽고 이 반전을 **언제쯤 예상할 수 있다**고 보십니까? 그 예상 타이밍이 **의도한 것**인지 확인합니다."
3. **기대 전략**
   - "이 전개는 장르 독자의 기대를 **충족** 합니까, **배신** 합니까, **우회** 합니까? 왜 이 전략을 선택했습니까?"
4. **대안 비교 후 선택 근거**
   - 대안 5개 중 작가가 선택한 것에 대해: "이 방법이 다른 네 개보다 우월한 이유는 **작품 고유의** 어떤 이유에서 옵니까?"

작가가 **'잘 모르겠다'** 또는 **'관습이라 그냥 썼다'** 로 답하면 🔴 플래그 + 대안 3안 제시. **'의식적 선택이고 이유 명확'** 이면 🟢.

## 책임
`bible/structure.md`, `story/plan.md`, `.work/scenes-ch*.md` 를 읽고 작가와 대화해 **의식적 선택 여부**를 검증:

1. 반전 예측도 — 독자가 1장을 읽고 3장의 반전을 맞출 확률 (작가 예상치 vs 비평가 추정치 비교)
2. 관습 의존도 — 이 장르 독자가 '이 다음 씬'을 얼마나 자주 봤는가
3. 대안 경로 — 같은 감정·기능을 다르게 구현할 방법 3

## 주의
- "독창성" 이 자체 목적은 아니다. 독자 기대를 부수는 것과 부서뜨린 것을 혼동하지 말 것.
- 'Grounded ending' 같은 장르 컨벤션은 의도적으로 유지 가능 — 이유를 적어라.
