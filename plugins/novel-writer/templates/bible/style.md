# 문체 가이드 (Style Bible)

> Style Linter (Gate G1) 가 이 파일과 `style-rules.json` 을 함께 읽어 검수.
> 집필 단계 진입 후 Read-only. 수정은 `/bible-unlock` 후에만.

## 서술 시점
- 3인칭 제한 시점 기본.
- 챕터 단위 POV 인물 고정. 교차는 `---` 섹션 구분자로 분리.

## 시제
- 주 서술: 과거형
- 회상·대과거: `~했었다` 형태 (층위 분리)

## 문장 리듬
- 평균 목표 길이 15~25자. 장면 태그(`<!-- scene: action -->` 등)에 따라 조정.
- 같은 길이 문장이 4회 이상 연속되지 않게.

## 어휘 — 금지·기피 (style-rules.json 자동 검출)
### 번역투 차단
`~의 것이다`, `~인 것이다`, `~로부터의`, `~에 의한`, `~을 가진다`, `~에 있어`, `~을 통해서`, `~에 다름 아니다`, `~지 않으면 안 된다`, `~할 필요가 있다` 등.

→ 한국어다운 능동·간결한 표현으로 치환.

### 구어 / 채팅체
`ㅎㅎ`, `ㅋㅋ`, `ㅠㅠ`, `???`, `!!!`, `.....` 모두 금지.

### 과부사 중첩 (한 문장 2개 이상)
`매우`, `정말`, `너무`, `아주`, `굉장히`, `엄청`, `되게`, `참`, `꽤` 중 둘 이상이 한 문장에 들어가면 경고.

## 금지 패턴 (AI slop 방지)
### AI 시작구
챕터·섹션 시작 문장에 다음 패턴 금지:
- `그날 아침`, `그날 밤`, `운명의`, `~는 생각했다`, `~는 깨달았다`
- `어쩌면`, `마침내 그`, `바로 그때`, `그 순간`
- `~에게 있어`, `어느 날 갑자기`, `모든 것이 시작된`

### 감정 클리셰 (현대 LLM 한국어 슬롭)
- `심장이 두근거렸다`, `숨이 멎는 듯했다`, `눈물이 흘러내렸다`
- `한숨을 내쉬었다`, `고개를 끄덕였다`, `어깨를 으쓱했다`, `미소를 지었다`

→ 행동·감각으로 구체화: "슬펐다" → "창밖을 오래 바라봤다"

### 직유 클리셰
`마치 ~처럼`, `~ 그 이상이었다`, `결코 잊을 수 없을` 패턴 경고.

## 대사 규칙
- 직접 인용: `" "`
- `"~라고 말했다"` 연속 2회 이상 금지. 동사 변형: 물었다 / 중얼거렸다 / 내뱉었다 / 속삭였다 / 되물었다 / 외쳤다
- `"큰 소리로 말했다"` 같은 대사 태그 수식 자제 — 지문·맥락이 더 강함

## 장면 태그 (scene override)
문장 상단 주석으로 임계값 조정:
```
<!-- scene: action -->     → 5~15자, 짧은 호흡
<!-- scene: dialogue -->   → 3~30자, 대사 중심
<!-- scene: lyrical -->    → 20~40자, 묘사·서정
<!-- scene: interior -->   → 15~35자, 내면 독백
```

## 한국어 어문규정 빈출 오류 (Proofreader 가 참조)
- `되요` → `돼요` / `안되` → `안 돼`
- `왠일` → `웬일` / `어떻해` → `어떡해`
- `뵈요` → `봬요`
- 동음이의 주의: 낳다(出産) / 낫다(better, heal) / 낮다(low)
- 맞히다(과녁) vs 맞추다(짝)

## 확장 규칙
`bible/style-rules.json` (프로젝트 루트 기준 상대 경로) 에 기계 판독용 규칙을 병행 유지한다. Style Linter 는 두 파일을 모두 읽는다.

작품 장르·작가 색깔에 따라 위 규칙을 자유롭게 추가·완화하라. 다른 장르 시작점은 `templates/genres/<장르>/style-rules.json` (예: `historic-noir`, `urban-fantasy`, `web-novel`) 참조.
