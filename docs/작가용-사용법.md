# 처음 쓰는 작가를 위한 안내서

> 이 문서는 **개발자가 아닌 작가**를 위해 쓰였습니다. 컴퓨터에 명령을 직접 입력하는 게 처음이어도 따라할 수 있습니다.

이 도구는 *Claude(클로드)* 라는 AI에게 **소설 집필을 도와달라고 부탁하는 방식**으로 작동합니다. Claude 는 우리가 글로 시키는 일을 하고, 그 과정에서 인물·세계관·문체의 일관성을 자동으로 점검해 줍니다.

---

## 0. 시작하기 전에 — 이게 뭐예요?

### Claude 가 뭐예요?
Claude 는 사람과 한국어로 대화하는 AI입니다. 우리가 "주인공의 어린 시절 상처는 어떤 모양일까?" 같은 질문을 하면, 같이 생각해 주고 정리해 줍니다.

### Claude Code 가 뭐예요?
Claude 를 **컴퓨터의 글자 화면(터미널)** 에서 쓰는 프로그램입니다. 채팅창처럼 말하면 답하지만, **파일을 직접 만들고 수정**할 수 있는 게 차이입니다 — 한 마디로, **우리가 시킨 대로 원고를 진짜 폴더에 저장해 줍니다**.

### novel-writer 플러그인이 뭐예요?
Claude Code 에 끼우는 **소설 작가용 부속품**입니다. 다음 두 가지를 자동으로 해줍니다.

1. **결정의 동반자** — 캐릭터·플롯·장면을 정할 때, Claude 가 답을 던져주는 게 아니라 **질문을 던져서 작가의 답을 끌어냅니다** (소크라테스식 대화).
2. **일관성 감시인 5명** — 챕터를 쓸 때마다 5명의 검수원이 자동으로 원고를 본 뒤 OK/재작성 판정을 내립니다. 이름이 챕터 1에서는 "수민", 챕터 5에서는 "수영"으로 변하는 식의 실수를 미리 잡아냅니다.

## 1. 설치 — 한 번만 하면 돼요 (15~30분)

### 1-1. 터미널 열기
"터미널" 은 Mac 의 기본 프로그램이에요. **글자로 컴퓨터에 명령**하는 창입니다.

1. 키보드에서 **`⌘ + Space`** (커맨드+스페이스) 를 누릅니다 → 화면 가운데에 검색창이 뜹니다 (스포트라이트).
2. **"터미널"** 이라고 한국어로 칩니다.
3. 까만 창이 열립니다. 이게 터미널입니다.

이 까만 창에 우리가 글자로 명령을 적으면, 컴퓨터가 그대로 실행합니다.

### 1-2. Claude Code 설치
터미널에 아래 한 줄을 **그대로 복사해서 붙여넣기** 합니다 (마우스 우클릭 → 붙여넣기, 또는 `⌘+V`).

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

엔터(`return` 키) 를 누릅니다. 글자가 한참 흘러간 뒤 끝나면 설치 완료.

### 1-3. Claude 에 로그인
같은 터미널에:

```bash
claude
```

입력 후 엔터. 처음이면 "로그인하시겠습니까?" 같은 안내가 뜹니다. 안내대로 따라가면 브라우저가 열려서 로그인 페이지가 나옵니다.

> **어느 계정으로 로그인하나요?**
> Claude.ai 에 가입한 계정 (이메일 + 비밀번호) 입니다. **무료 / Pro / Max / Team / Enterprise** 모두 가능. Pro 이상 권장 (장편 한 편을 쓰기에 무료 한도는 부족할 수 있음).

로그인 끝나면 터미널에 "환영합니다" 비슷한 메시지가 보입니다.

### 1-4. novel-writer 플러그인 설치
Claude 에게 이렇게 칩니다:

```
/plugin marketplace add hansangho/novelai
```

엔터. 잠시 후 "마켓플레이스가 추가되었습니다" 같은 메시지.

이어서:

```
/plugin install novel-writer@hans-novel-tools
```

엔터. "설치되었습니다" 메시지가 나오면 완료.

> **에러가 나면?**
> 파란 메시지·빨간 메시지가 보여도 당황하지 마세요. 메시지 전체를 복사해서 Claude 에게 한국어로 붙여넣고 "이거 어떻게 해결해?" 라고 물어보면 알려줍니다.

### 1-5. Claude 종료
잠깐 종료하려면 터미널에 `exit` 를 치고 엔터, 또는 `Ctrl+C` 두 번. 다시 열려면 `claude` 를 치면 됩니다.

## 2. 첫 작품 폴더 만들기 (5분)

소설 한 편마다 **별도 폴더** 를 씁니다. 다른 작품과 섞이지 않도록.

### 2-1. 폴더 만들기
터미널에서:

```bash
mkdir -p ~/Documents/내소설들/내첫소설
cd ~/Documents/내소설들/내첫소설
```

설명:
- `mkdir -p ~/Documents/내소설들/내첫소설` → "내 문서 안에 '내소설들/내첫소설' 폴더를 만들어라"
- `cd ~/Documents/내소설들/내첫소설` → "그 폴더 안으로 들어가라"

원하는 이름·위치로 바꿔도 됩니다 (예: `~/Desktop/소설/광주의겨울`).

### 2-2. Claude 시작 + 스캐폴딩
같은 터미널에서:

```bash
claude
```

Claude 가 켜진 뒤:

```
/init-novel
```

→ 폴더 안에 `bible/`, `state/`, `timeline/`, `story/` 같은 하위 폴더와 빈 템플릿 파일들이 자동으로 들어옵니다. 이게 작품의 **뼈대**입니다.

> **각 폴더가 뭐예요?**
>
> | 폴더 | 비유로 설명 |
> |------|-------------|
> | `bible/` | **작품 사전** — 캐릭터·세계관·문체·금지선. 한 번 정하면 잘 안 바뀜. |
> | `state/` | **챕터별 노트** — 챕터 1 끝났을 때 누가 어디 있었지? 같은 메모. 자동 작성. |
> | `timeline/` | **사건 일지** — 1923년 1월 15일에 무슨 일이 있었는지 시간순. 자동 작성. |
> | `story/` | **실제 원고** — 챕터 1, 2, ... 의 본문. |
> | `.work/`  | **임시 작업장** — 검수 통과 안 한 초고. 자동 정리. |
> | `.session/` | **세션 메모** — 어디까지 작업했는지. 자동. |

## 3. 캐릭터·세계관 만들기 — 소크라테스 모드 (1~3시간)

이 도구의 가장 특별한 점은, **Claude 가 "이 캐릭터는 이렇게 하면 어때요?" 라고 답을 주지 않는다**는 것입니다. 대신:

> "이 인물의 어린 시절 가장 오래된 상처는 어떤 모양입니까?"
> "이 인물이 **절대 하지 않을** 한 가지 행동은 무엇입니까?"

같은 **질문을 던져서 작가가 답을 발견**하게 합니다. 답을 듣고 다음 질문을 쌓아 갑니다.

### 3-1. 주제부터 — 무엇을 쓰고 싶나요?
Claude 에게 한국어로 그냥 말하세요:

```
역사 느와르 장르로, 1920년대 경성을 배경으로 한 소설을 쓰고 싶어. 주제 같이 찾아줘.
```

Claude 가 **theme-scout** 라는 부하 직원을 호출해 질문을 던집니다:
- "최근 며칠 안에 반복해서 떠오르는 이미지나 감정이 있나요?"
- "당신이 쓰기 두려운 이야기는 무엇입니까?"

답하다 보면 본인도 몰랐던 진짜 주제가 드러납니다. 3~5개 질문 후에 Claude 가 정리해 줍니다.

### 3-2. 캐릭터 만들기
```
주인공 한 명을 같이 만들어보자.
```

Claude 가 **character-architect** 를 호출해 7단계 질문을 던집니다:
1. 가장 오래된 상처
2. 스스로에게 인정하지 못하는 욕망
3. 강점이 약점으로 바뀌는 지점
4. 절대 하지 않을 행동
5. 타인의 오해
6. **지식 경계 — 알 수 있는 / 알 수 없는 것** ← 가장 중요!
7. 신뢰하는 한 사람과 그 배신 방식

답하면서 인물이 살아납니다. 끝나면 `bible/characters/김도현.md` 같은 파일이 자동 작성됩니다.

> **"지식 경계" 가 왜 중요한가요?**
> 챕터 7에서 주인공이 알게 되는 비밀을 챕터 3에서 모르게 해야 합니다. 이걸 명시해 두지 않으면, 나중에 Claude 가 "아 이 인물 이거 이미 알고 있었지" 하고 잘못 쓰게 됩니다.
> 우리가 **챕터 7에서 알게 됨** 이라고 설정해 두면, 챕터 3 검수 때 "이 인물이 이걸 아직 모를 텐데?" 하고 자동으로 잡아 줍니다.

### 3-3. 세계관·시대고증
같은 방식으로:

```
세계관 같이 짜자. 1920년대 경성 배경에서 무엇이 가능하고 무엇이 불가능한지 정해두고 싶어.
```

`bible/universe/rules.md` (작품 안의 규칙), `historical-facts.md` (역사 고증) 가 채워집니다. 예: "1923년 조선에는 휴대전화 없음" 같은 사실. 챕터에서 휴대전화 언급하면 자동 차단됩니다.

### 3-4. 전체 구조
```
전체 플롯 같이 잡아보자. structure-architect 호출해줘.
```

Claude 가 **결말 이미지부터 거꾸로** 묻습니다:
- "마지막 장면에서 카메라가 마지막까지 머무는 지점은 어디입니까?"
- "중간 지점(약 절반) 에서 되돌아갈 수 없는 사건은 무엇입니까?"
- "오프닝이 독자에게 하는 약속 중 의도적으로 배신할 약속은?"

답하다 보면 챕터 1, 2, 3, … 의 윤곽이 잡힙니다. `story/plan.md` 에 자동 저장.

### 빠르게 가고 싶으면
질문이 많이 느껴지면, 어느 단계에서든 Claude 에게:

```
빠른 드래프트 모드로 옵션 3개만 줘.
```

라고 말하면 질문 없이 바로 후보를 제시합니다.

## 4. 본격 집필 — 챕터 1 (1~2시간)

### 4-1. Bible 잠그기
충분히 설정이 짜여졌으면:

```
/bible-lock
```

이제부터 `bible/` 폴더의 설정 파일들은 **잠긴 상태**가 됩니다. 실수로 바꾸지 않도록 안전 장치. 정말 바꾸고 싶다면 `/bible-unlock "이유"` 로 풀 수 있습니다.

### 4-2. 첫 챕터 쓰기
```
/write-chapter 1
```

Claude 가 알아서:
1. 챕터 1 의 장면별 계획을 짭니다 (Scene Planner).
2. 초고를 씁니다 (Chapter Writer).
3. **5명의 검수원을 자동 호출**:
   - **G1 문체 검수** — 금지 어휘·과부사 등
   - **G2 캐릭터 검수** — 인물이 자기답게 행동하는가, 아직 모를 사실을 말하지 않는가
   - **G3 연속성 검수** — 시간·공간·세계관 규칙 위반
   - **G4 독창성 검수** — 너무 뻔한 문장 표시 (선택적)
   - **G5 종합 검수** — 위 4 명의 의견을 종합
4. 검수 통과 못 한 부분이 있으면 **자동 재작성** (최대 3번까지).
5. 통과하면 작가 확인 요청.

### 4-3. 결과 보기
모든 검수가 끝나면 Claude 가 보고합니다:

> "챕터 1 검수 결과: G1 통과, G2 통과, G3 통과, G4 권고 3건 (작가 판단), G5 통과. 확정하시겠습니까?"

OK 면:

```
/finalize 1
```

→ `story/chapters/chapter-01.md` 가 확정 저장됩니다.

### 4-4. 검수에 실패했다면?
Claude 가 어디가 왜 실패했는지 친절히 알려줍니다. 예:

> "G2 캐릭터 검수 실패. 김도현이 챕터 3 이전에 알 수 없는 사실을 챕터 1에서 언급했습니다. (45행 '딸이 의심한다')"

이런 안내를 보고:
- **그대로 자동 재작성** 시키기 → 다음 시도(v2) 까지 자동
- **본인이 직접 수정** 하고 싶으면 → 파일을 열어 고친 뒤 검수만 다시 돌리기 (`/run-gate G2 1`)

## 5. 매일 작업 패턴

### 작업 시작
터미널에서:
```bash
cd ~/Documents/내소설들/내첫소설
claude
```

작업 이어쓰기:
```
/resume
```

→ 어디까지 했는지 정리해 줍니다.

### 다음 챕터
```
/write-chapter 2
```

### 챕터 끝 상태 확인
```
/state 5
```

→ 챕터 5 끝났을 때 누가 어디 있고 무엇을 알고 있는지.

### 사건 검색
```
/timeline
```
또는 인물 이름으로:
```
/timeline 김도현
```

### 미회수 떡밥 확인
```
/timeline --unresolved
```

### 작품 통계
```
/metrics
```

→ 챕터 수·평균 분량·재작성 평균 횟수 등.

## 6. 자주 막히는 지점

### "Claude 가 영어로 답해요"
한국어로 한 줄 추가하세요:
```
한국어로 답해줘.
```

### "검수에서 자꾸 떨어져요"
- **정보 누수 (G2)**: 캐릭터의 "지식 경계" 가 모호한 게 원인. `bible/characters/<이름>.md` 의 "알 수 없는 것" 목록을 다시 채워보세요.
- **세계관 위반 (G3)**: 챕터에 시대 오류가 있는 것. `bible/universe/historical-facts.md` 와 충돌하는 부분 수정.
- **문체 오류 (G1)**: 너무 엄격하면 `bible/style.md` 의 임계값을 완화. 또는 그 장면에 "긴장 장면" 태그 부여.

### "Claude 가 너무 많이 물어봐요"
```
빠른 드래프트 모드로.
```
또는:
```
소크라테스 건너뛰어. 그냥 옵션 3개 줘.
```

### "지금 어떤 상태인지 모르겠어요"
```
/resume
```
또는:
```
지금까지 우리 뭐 했지? 정리해줘.
```

### "캐릭터 이름을 바꾸고 싶어요"
이미 챕터를 몇 개 썼다면 Bible 이 잠겨 있습니다. 이렇게:
```
/bible-unlock "주인공 이름을 김도현에서 박진우로 변경"
```

→ Claude 가 자동으로 모든 챕터·timeline·state 파일에서 영향받는 부분을 찾아 보여줍니다.
수정 끝나면:
```
/bible-lock
```

### "이전에 쓴 챕터를 다시 보려면?"
폴더에서 직접: `~/Documents/내소설들/내첫소설/story/chapters/chapter-01.md` 를 텍스트 편집기 (메모장·VS Code·Pages 등) 로 열면 됩니다.

또는 Claude 에게:
```
챕터 3 본문 보여줘.
```

### "Claude 를 끄면 작업이 사라지나요?"
**아니요.** 모든 파일이 폴더에 저장됩니다. 다음에 같은 폴더에서 `claude` 다시 실행 → `/resume` 하면 계속됩니다.

### "다른 컴퓨터에서도 작업하려면?"

## 8. 안전 장치 — 실수해도 돌이킬 수 있어요

### 자동 백업
1. **모든 파일은 디스크에 저장됨** — Claude 가 갑자기 종료돼도 사라지지 않음.
2. **Bible 잠금** — 집필 중 실수로 설정을 바꾸지 못하게 함.
3. **Timeline 추가만 가능** — 과거 사건을 실수로 지우지 않음.
4. **드래프트 / 확정본 분리** — 검수 실패한 글은 `.work/` 에 따로 저장. 확정본 (`story/chapters/`) 은 안전.

### Time Machine 권장
Mac 의 **Time Machine** 백업을 켜두시면 더 안전합니다 (시스템 환경설정 → 일반 → Time Machine).

### 불안하면 가끔 폴더 복사
중요한 시점마다 (예: 챕터 5 끝났을 때) 폴더 전체를 다른 위치에 복사해 두면 됩니다:
```bash
cp -r ~/Documents/내소설들/내첫소설 ~/Documents/내소설들/내첫소설_백업_챕터5까지
```
